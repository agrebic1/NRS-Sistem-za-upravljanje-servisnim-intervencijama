----> TABELA 
| Stakeholder | Uloga | Interes | Očekivanja | Uticaj na sistem | Prioritet komunikacije |
|-------------|-------|---------|------------|------------------|------------------------|
| Klijent | Prijavljuje kvar, unosi i ažurira zahtjev, prati status intervencije, ocjenjuje uslugu i podnosi žalbe ili zahtjev za ponovnu intervenciju. | Brzo rješavanje problema, transparentan status, mogućnost korekcije zahtjeva i historija svojih prijava. | Jednostavna prijava, jasni statusi (čekanje, odobreno, završeno), pravovremene notifikacije, mogućnost eskalacije. | | |
| Dispečer | Pregleda zahtjeve, određuje prioritet, dodjeljuje serviserima, prati tok intervencije i potvrđuje završetak. | Efikasna raspodjela resursa, pregled svih aktivnih intervencija, kontrola toka rada. | Centralizovan pregled zadataka, filtriranje po prioritetu/lokaciji, podrška za dodjelu više servisera, validne statusne tranzicije. | | |
| Serviser | Prihvata ili odbija zadatke, izvršava intervencije, ažurira statuse i evidentira rad (vrijeme, materijal, napomene). | Jasni zadaci, realni rokovi, evidencija rada i jednostavno ažuriranje statusa na terenu. | Mobilna dostupnost, jednostavan unos podataka, mogućnost prijave problema i traženja pomoći. | | |
| Glavni serviser | Koordinira tim na terenu, raspoređuje podzadatke, validira rad i zatvara intervenciju (ako postoji više servisera). | Kontrola nad timom, pregled svih aktivnosti članova, odgovornost za završetak. | Hijerarhijska prava, pregled timskih zadataka, mogućnost finalnog zatvaranja intervencije. | | |
| Menadžer | Definiše SLA pravila, prati performanse (KPI), upravlja eskalacijama. | Efikasnost sistema, produktivnost timova, kvalitet usluge. | Tačni i agregirani izvještaji (vrijeme odziva, rješavanja, broj intervencija), pregled SLA kršenja. | | |
| Administrator | Upravljanje korisnicima i pristupnim pravima (RBAC), konfiguracija sistema i održavanje podataka. | Stabilnost sistema, sigurnost, kontrola pristupa. | Jednostavna administracija, audit log, kontrola prava pristupa, pouzdanost sistema. | | |
| CEO/Vlasnik | Strateški nadzor i donošenje poslovnih odluka. | ROI, efikasnost poslovanja, optimizacija troškova. | Sažeti izvještaji (trendovi, performanse, troškovi), bez operativnih detalja. | | |
| HR/Finansije | Praćenje radnih sati, troškova i produktivnosti zaposlenih. | Tačna evidencija rada i materijala. | Precizni podaci za obračun plata, analiza učinka. | | |
| IT tim | Održavanje sistema, tehnička podrška, razvoj i skaliranje. | Stabilnost, sigurnost, performanse i skalabilnost. | Jasna arhitektura, auditabilnost, logovi, mogućnost proširenja (više lokacija/timova). | | |
